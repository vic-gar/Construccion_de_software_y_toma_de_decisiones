Apartado de Preguntas

1. ¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una infraestructura que se utiliza para conectar algun dispositivo con cualquier otro en cuaquier parte del mundo, mientras que la World Wide Web es una colección de información a la que se accede mediante Internet (Global Connect, 2024).

2. ¿Cuáles son las partes de un URL?

Un esquema (HTTP o HTTPS), un nombre de host, número de puerto, la ruta y una cadena de consulta, como el siguiente ejemplo:

scheme://hostname:port/path?query

(International Bussines Machines, IBM, 2026).

3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

  • GET: se utiliza para recuperar datos del servidor. Es un método de solo lectura, por lo que no existe riesgo de que los datos se modifiquen o se corrompan.
  • HEAD: el servidor devuelve solo las cabeceras (headers) y no el cuerpo de la respuesta.
  • POST: envía datos al servidor y crea un nuevo recurso, el cual queda subordinado a otro recurso padre.
  • PUT: se utiliza con mayor frecuencia para actualizar un recurso existente.
  • PATCH: es muy similar al método PUT, la diferencia radica en que el cuerpo de la solicitud solo necesita contener los cambios específicos realizados al recurso.
  • DELETE: se utiliza para eliminar un recurso especificado por su URI.

(Stack Overflow, 2020).

4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Se usa POST para enviar formularios como login de usuario y contraseña, ya que transmite datos sensibles en el cuerpo de la petición (no en la URL como GET), mejorando seguridad y permitiendo mayor volumen de datos (Sanchez, J., 2026).

5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

GET se utiliza al ingresar una URL en el navegador, ya que solicita el recurso sin alterar datos del servidor (Sanchez, J., 2026).

6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

No sucedió ningun error, el código 200 OK significa que la solicitud tuvo éxito.

7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Si es responsabilidad del desarrollador parcalmente debido a que este código indica que el servidor no pudo encontrar el contenido que se le solicita, por ende se deben de verificar rutas o redirigir. También puede ser culpa del usuario si escribió mal la URL

8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Si es su responsabilidad debido a que este código indica que hubieron fallos en el servidor o en configuraciones externas.

9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Son aquellos que el W3E ha designado como desactualizados y que ya no pueden usarse ya que pueden no ser soportados en futuras actualizaciones o navegadores (Coyier, C., 2007)

10. ¿Qué componentes de estructura y estilo tiene una tabla?

Aquí hay algunos ejemplos de componentes de estructura y estilos para tablas:

  • Table (contenedor), tr (filas), td/th (celdas), thead/body/tfoot (secciones cabeza/cuerpo/pie), caption (título).
  • Atributos como border, cellpadding, cellspacing.
(Mary, 2026)

11. ¿Cuáles son los principales controles de una forma HTML5?

Los principales controles son los siguientes: input (text, password, email, date, checkbox, radio, file), select (listas desplegables), textarea (texto multilineal), button, con validaciones como required, pattern (MDN, Resources for Developers, by Developers, s.f.)

12. Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información? ¿Cuál es el ciclo de desarrollo de sistemas de información?

  • Ciclo de vida: planificación, desarrollo, implementación, operación, mantenimiento y retiro de un sistema completo.
  • Ciclo de desarrollo: requisitos, diseño, codificación, pruebas, implementación y mantenimiento.
(ServiceNow, s.f.)
