Preguntas y respuestas

1. ¿Qué ventajas ofrecen las variables CSS sobre los valores hardcodeados?

Las variables CSS ayudan a mantener el código más ordenado y fácil de modificar, ya que permiten reutilizar valores sin tener que cambiarlos en cada parte del archivo, haciendo que el diseño sea más consistente y práctico de mantener.

2. ¿En qué casos conviene declarar variables CSS fuera del selector :root?

Es mejor hacerlo cuando las variables solo se van a usar dentro de un componente o sección específica, ya que así se limita su alcance y se evita afectar otras partes del diseño.

3. ¿Cómo se pueden utilizar las variables CSS para implementar un sistema de temas (como modo oscuro)?

Una forma común es definir variables en :root y luego sobrescribirlas en una clase como .dark, de esta manera, al cambiar la clase en el body, los colores y estilos cambian automáticamente.

4. ¿Qué sucede cuando intentas usar una variable CSS que no ha sido declarada?

En ese caso, el navegador no puede aplicar correctamente la propiedad, por lo que puede ignorarla o utilizar un valor de respaldo si este fue definido previamente.

5. ¿Cuál es la diferencia entre los enfoques mobile-first y desktop-first?

El enfoque mobile-first consiste en diseñar primero para dispositivos móviles y después escalar a pantallas más grandes, mientras que desktop-first hace lo contrario, iniciando desde pantallas grandes y adaptándose hacia tamaños más pequeños.

6. ¿Qué breakpoints son los más comunes y por qué?

Generalmente se utilizan breakpoints como 480px, 768px, 1024px y 1280px, ya que corresponden a tamaños típicos de celulares, tablets y computadoras, lo que facilita adaptar el diseño a diferentes dispositivos.

7. ¿Cómo afecta el viewport meta tag al diseño responsivo?

El viewport permite que el diseño se ajuste correctamente al ancho del dispositivo, lo cual es fundamental para que las páginas se vean bien en celulares.

8. ¿Qué técnicas además de media queries existen para crear diseños responsivos?

Además de media queries, se pueden utilizar herramientas como Flexbox, CSS Grid, unidades relativas y funciones modernas como clamp(), que ayudan a crear diseños más flexibles.

9. ¿Cómo puedes identificar qué regla CSS está siendo aplicada cuando hay múltiples selectores?

Esto se puede analizar utilizando las herramientas de desarrollo del navegador, donde se muestran las reglas aplicadas y cuáles han sido sobrescritas.

10. ¿Qué información proporciona la pestaña Computed en DevTools?

La pestaña Computed muestra los valores finales que el navegador aplica a cada propiedad, considerando la cascada, herencia y prioridades del CSS.

11. ¿Cómo puedes simular diferentes dispositivos sin tener acceso físico a ellos?

Se puede hacer mediante el modo responsivo en las herramientas de desarrollo del navegador, el cual permite visualizar cómo se vería la página en distintos dispositivos.

12. ¿En qué situaciones es preferible usar Bootstrap sobre Tailwind CSS y viceversa?

Bootstrap es útil cuando se necesita desarrollar rápido con componentes ya definidos, mientras que Tailwind es mejor cuando se busca mayor personalización y control sobre el diseño.

13. ¿Qué significa que Tailwind sea "utility-first"?

Significa que utiliza clases pequeñas que cumplen una sola función, lo que permite construir el diseño directamente en el HTML sin depender de estilos predefinidos complejos.

14. ¿Cómo afecta el uso de un framework CSS al tamaño final del bundle?

El tamaño puede aumentar si no se eliminan estilos que no se utilizan, aunque actualmente existen herramientas que permiten optimizar el CSS y reducir este problema.

15. ¿Es posible combinar múltiples frameworks CSS en un mismo proyecto? ¿Por qué sí o por qué no?

Aunque sí es posible, no es recomendable, ya que puede generar conflictos entre estilos, redundancia en el código y un mayor peso en el proyecto.
